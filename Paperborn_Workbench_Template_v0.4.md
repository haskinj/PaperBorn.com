# Paperborn Workbench Template v0.4

**Personal workflow draft — Jesse Haskin / Paperborn Project / GlitterGhost Academy**
**Date: 2026-05-16**
**Supersedes: v0.3**

---

# What changed from v0.3

v0.4 emerged from testing against:

* dense meta-analyses
* abstract topology papers
* QCA / fusion-category theory
* clinical oncology synthesis papers
* deliberately boring methodology papers

The major discovery:

> **Humanoid bias was suppressing the actual soul of the paper.**

The strongest Paperborns were often the least human:

* forest-plot swarms
* lattice engines
* braided theorem organisms
* geological strata-creatures
* citation-web hives
* probabilistic fog entities
* topological scar-machines

v0.4 therefore introduces:

* **Body Plan Liberation**
* **Method-Derived Anatomy**
* **Evidence-Encoded Morphology**
* **Non-Humanoid Priority Rules**
* **Diagram-as-Anatomy Systems**

---

# New First Rule (v0.4)

> **The Paperborn’s anatomy must emerge from the paper’s structure.**
>
> Do not default to humanoid form unless the paper itself suggests:
>
> * agency
> * personality
> * embodiment at human scale
> * dialogue
> * ritual
> * social identity
>
> A Paperborn should not look like:
> “a person wearing the paper.”
>
> It should look like:
> “the paper became physically real.”

# Core philosophical shift

v0.3 often produced:

```text
cool fantasy character inspired by research
```

v0.4 aims for:

```text
epistemological organism
```

The creature is not a mascot.

It is:

* the method made flesh
* the uncertainty made topology
* the dataset made anatomy
* the theorem made ecology
* the statistical structure made behavior

* a defining chart
* network diagram
* lattice
* flowchart
* forest plot
* architecture schematic
* braid
* filtration
* tensor structure

…then that diagram should influence:

* skeleton

```text
structural anatomy
```


```text
paper-themed costumes
```

Not Paperborn.

# NEW: Material Stack System

v0.3 introduced:

```text
core material
```

v0.4 expands this into:

## Material Stack

| Layer               | Purpose                              |
| ------------------- | ------------------------------------ |
| Core Material       | Fundamental conceptual substance     |
| Structural Material | Skeleton / load-bearing system       |
| Surface Material    | Texture / interface                  |
| Failure Material    | Represents uncertainty or limitation |

Example:

```text
Core: frosted borosilicate glass
Structure: copper lattice
Surface: condensation film
Failure: fractured quartz seams
```

This creates:

* richer physical logic
* stronger visual coherence
* better image generation consistency

# NEW: Constraint Fidelity Rule

The strongest Paperborns preserve:

* limitations
* uncertainty
* scope
* assumptions
* incompleteness

A Paperborn should not become:

```text
generic omnipotent god
```

If the paper:

* only works under constraints
* applies locally
* assumes finite conditions
* depends on approximations

…then the entity must physically encode those dependencies.

# NEW Failure Test

Before finalizing a Paperborn, ask:

## Could this creature still exist if all labels/text vanished?

If NO:
the embodiment is too shallow.

The Paperborn should still visually imply:

* its method
* its structure
* its constraints
* its field

without readable text.
